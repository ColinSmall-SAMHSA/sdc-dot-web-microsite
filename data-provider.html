Data Provider

Secure Data Commons (SDC) enables collaborative but controlled integration and analysis of research data at the moderate sensitivity level (PII & CBI).

Within the SDC a Data Provider can provide data in near-real-time throughout their projects. This allows Data Providers to develop common data formats and fix issues during testing, and it enables project evaluators to start their analysis earlier, build upon each otherâ€™s work, and access new sources of data.

Data Providers define the terms of data access and can grant or deny access to specific users or groups. Additionally Data Providers can grant or deny access to what type of derived data is exported or copied from the system. Raw data cannot be copied or exported from the system. The SDC offers authorized and controlled access to individual datasets as well as the metadata associated to those datasets.

Supports three types of data transfer:

  1. Real-Time (e.g. Streaming)
  2. Batch Uploads (e.g. Daily or weekly uploads)
  3. Ad-hoc Uploads (Occasional uploads)

Secure Data Commons Features

Robust cloud-based infrastructure for storage and management of data

  • Both real time ingest streams as well as batch uploads
  • Strong export controls to take derived data out of the SDC
  • Set levels of data curation and warehousing to support analysis

Built-in infrastructure for data analysis

  • Pre-established analysis workstations with open source tools
  • Ability to import and share code between analysts

Strong user management controls

  • Multi-factor authentication and PIV-Card Integration (in progress)
  • Workflow for data import/export into and out of the SDC
  • Control access to specific data, data types, and metadata by user groups and individual users

Effective Project and Team Collaboration

  • Project-level controls and teams
  • Shared team resources
